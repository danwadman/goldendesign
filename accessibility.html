Accessibility

A few examples of common acessiblity mistakes

Contrast

WCAG 2.1 - SC 1.4.3 Contrast (Minimum)

The visual presentation of text and images of text has a contrast ratio of at least 4.5:1, except for the following: (Level AA)

  • Large Text: Large-scale text and images of large-scale text have a contrast ratio of at least 3:1
For example
  • Background colour: #f1f1f1;
  • Text color: #000000;

This would meet a contrast ratio of 18.59:1

The minimum needed to meet compliance with font size 16px or less is 4.5:1

https://webaim.org/resources/contrastchecker/

Styling

WCAG 2.1 - SC 1.3.1 Info and Relationships

Always try to use html's semantic elements to convey meaning rather than using presentation techniques.

For example
  • Use html heading tags to denote sections.
  • Use lists instead of table markup with images of bullet point to show lists items

Always try to seperate style from layout.

Images

WCAG 2.1 - SC 1.1.1 Non-text content

Make sure every image has an alt text and that it is relevant to the image and not a default text.

Keyboard

WCAG 2.1 - SC 2.1.1 Keyboard

Wherever possible, content can be operated through a keyboard or keyboard interface. For example can all of the admin menu be accessed using just the keyboard?

Guidance

Everything should be created with accessibility in mind.

All elements should meet WCAG 2.1 - Level AA
